Dataset

Overview & Source

Our project uses the Drug Consumption dataset compiled by Fehrman et al. (2015). The database contains records for 1,885 respondents. For each person we observe:

  • Demographics: age band, gender, education level, country, ethnicity
  • Personality measurements: Big Five (NEO-FFI-R), impulsivity (BIS-11), and sensation seeking (ImpSS)
  • Self-reported use of 18 legal and illegal drugs plus one fictitious control drug (Semeron)

All input attributes were originally categorical and then quantified so that they can be treated as real-valued features. In the version we use (Drug_Consumption.csv), some variables have been converted back into readable categories (for example, age bands like ‚Äú18‚Äì24‚Äù and education levels).

Project-Specific Data Preparation

The Drug_Consumption.csv file included with this site is the working dataset we use in our analyses. It is adapted from the original quantified data with the following aims:

  • Keep personality scores (Nscore, Escore, Oscore, Ascore, Cscore, Impulsive, SS) as real-valued features.
  • Present demographics such as Age, Education, Country, and Ethnicity in human-readable categories.
  • Preserve the original CL0‚ÄìCL6 labels for each drug so that we can flexibly derive both binary and ordinal outcomes.
  • Standardize variable names and labels to align with our codebook and figures.

Because all personality traits are numeric and drug outcomes can be expressed as ordered or binary variables, the dataset is well-suited to:

  • Descriptive statistics and group comparisons (e.g., users vs. non-users)
  • Correlation and association analyses between traits and usage
  • Exploratory modeling of risk profiles (although this project focuses on EDA rather than predictive modeling)

Cleaned Analysis Dataset

For all figures and summaries in this project we use a cleaned analysis file, cleaned_drug_data.csv. After removing overclaimers (those who said they used Semer the fictious drug) and removing the Semer column itself, this dataset has a sample of 1,876 respondents and 31 variables. Our data cleaning process standardizes the coding to make analysis easier.

  • Age is stored as numeric codes representing ordered age bands.
  • Gender is stored as a binary variable representing gender categories (0 for male and 1 for female).
  • Each drug outcome is recorded as an integer 0‚Äì6, corresponding to the original CL0‚ÄìCL6 recency scale.
  • Personality traits remain continuous scores on their original scales.
  • Demographic fields such as education, country, and ethnicity are kept as readable categorical labels.
  • The cleaned file contains no missing values, so all respondents are retained for EDA.

This cleaned version is the starting point for all descriptive statistics, cross-tabulations, and visualizations shown on the other pages of the site.
